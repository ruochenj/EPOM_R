Introduction

Pacakge EPOM is a package developed corresponding to the paper Epigenome overlap measure (EPOM) for comparing tissue/cell types based on chromatin states. The package depends on rtracklayer in bioconductor in order to read the bigwig file into R. It also depends on the parallel package in order to speed up the computation.

source("https://bioconductor.org/biocLite.R")
biocLite("rtracklayer")
library(parallel)
library(readr)

Functions

1.epom()

epom() is the main function of the package, it takes 13 parameters. bw_list is a vector that containing all the full paths of bigwig files. bed_list is a vector that containing all the full paths of bed files. bd_header and bed_sep are used to read in the bed file using read.table. cell_type is a vector that containing all the cell types according to each cell. histmark and state_name are labels for the files to be saved.

An example for the bw_list, each element should be the full path.

bw_list <- c("C:/E001-H3K4me1.imputed.pval.signal.bigwig",
"C:/E002-H3K4me1.imputed.pval.signal.bigwig",
"C:/E003-H3K4me1.imputed.pval.signal.bigwig",
"C:/E004-H3K4me1.imputed.pval.signal.bigwig",

"C:/E007-H3K4me1.imputed.pval.signal.bigwig",
"C:/E008-H3K4me1.imputed.pval.signal.bigwig")

An example for the bed_list, each element should be the full path.

It is worth noticing that the bw_list, bed_list and cell_type all have the same length and they match each other correspondingly.

An example for the hismark.

histmark = "H3K4me1"

An example for state. Here Chromatin state is identified as table 1 in the paper. According to the paper state 10 to 18 correspond to the enhancer state.

state = 10:18

An example for state_name.

state_name = "enhancer"

in_dir indicates the directory for files that can be read in. An example for in_dir. The default value is NULL. If you don’t have any previously runned files, you can ignore this argument.

in_dir = "C:/Users/username/Desktop/read_in"

out_dir indicates the output directory. An example for out_dir. In the output directory, you can find RData files saved for each step. The final result will also be saved in this directory. The default value for out_dir is the current directory of the source file. If there’s no input for out_dir argument, the output files will be saved in the source file directory.

out_dir = "C:/Users/username/Desktop/output"

Since variables alpha_1, alpha_2, cores and m are given default values, some brief descriptions will be given here. alpha_1 is the threshold used in the anova test procedure. alpha_2 and m are the thresholds used in the t test procedure. cores are used to parallelize the computation, try to aviod using all the cores on the machine.

epom(bw_list, bed_list, bd_header = FALSE, bed_sep = "\t", cell_type,
  histmark, state, state_name, in_dir = in_dir, save = TRUE, out_dir = out_dir, alpha_1 = 1e-10, cores = 16,
  alpha_2 = 0.01, m = 13)

The result will be a matrix containing the matrix of the epom scores.

2.Individual Function Usage

If user want to perform their own analysis and just employ a portion of the functions in EPOM. The following example can be a guide.

#single bigwig file reading in
bw1 <- import.bw(C:/E001-H3K4me1.imputed.pval.signal.bigwig)
#bed file reading in
bed1 <- as.data.frame(read.table('E001_25_imputed12marks_stateno.bed',header = FALSE, sep='\t',stringsAsFactors=TRUE, quote=''))
#align bw file and bed file
#transform bw1 to 200 bp
matrix_gen_1 <- matrix_bp_trans(bw1, 200)
#get the index of the 200 bp matrix according to the bed file information and state information.
index <- index_num4state(matrix_gen, bed_1, state)

After the previous step, we will have a matrix called matrix_gen_1 containing signal of E001 cell but the width is 200bp. We will also get a vector of index indicating the selected regions according to enhancers in the cell. Then we will take the union of all the selected index regions among all the cells. The union will be the candidate select region.

More specifically, if we have 124 cell samples, we will generate a matrix called matrix_gen containing 3+124 = 127 columns, the first three columns record the information of “chromatin state”, “begin” and “width” as in the big wig file. For the rest 124 columns, each column will correspond to the signal in 200bp.

We will also have a vector called row_select that is the union of the indecies selected according to the enhancer region of each sample cell.

The next step is to do the anova test for each region.

l = anova_select3(cell_type, row_select, matrix_gen, state = "enhancer",
  alpha_1 = 1e-10, in_dir = in_dir, save = TRUE, out_dir = out_dir, cores = 16)

Then do the t test for each region.

l = t_select3(l$select_reg, l$mat_anova, cell_type, state = "enhancer",
  alpha_2 = 0.01, m = 13, in_dir = in_dir, save = TRUE, out_dir = out_dir, cores = 16)

Then do the epom score calculation.

table = epom_score(l_t, cell_type, save = TRUE, out_dir = out_dir)

Finally, we can get a table containing all the epom score information.
